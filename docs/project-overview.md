# RepIt – Workout Planning & Tracking Application

## Overview

RepIt is a full-stack workout planning and tracking application designed to help users create structured fitness routines, log workouts, and monitor progress over time. 

This project was built as a product-focused case study to simulate a real-world Agile delivery environment, combining structured requirements, AI-assisted development, and modern version control practices.

---

## Problem Statement

Many individuals lack a simple and intuitive way to plan workouts and consistently track their fitness progress. Existing solutions are often overly complex, lack personalization, or fail to provide a streamlined experience for both planning and tracking workouts.

---

## Solution

RepIt provides a lightweight and structured platform that enables users to:

- Create and manage personalized workout plans  
- Browse and select exercises  
- Log completed workouts  
- Track performance and progress over time  

The application focuses on simplicity, usability, and structured data capture to support long-term consistency and improvement.

---

## Approach

This project was developed using an Agile methodology to simulate a real product delivery lifecycle:

- **Project Management:** Jira used to define epics, user stories, acceptance criteria, and sprint planning  
- **Development:** Replit AI leveraged to implement features based on structured requirements  
- **Version Control:** GitHub used to manage code, track commits, and maintain traceability  
- **Workflow Integration:** Jira and GitHub integrated to link requirements directly to implementation  

Each feature was developed through a full cycle of:
1. Requirement definition  
2. Sprint planning  
3. AI-assisted implementation  
4. Validation against acceptance criteria  
5. Delivery tracking via commits  

---

## MVP Scope

The initial version of RepIt focuses on delivering core functionality:

- User authentication (registration, login, logout)  
- Workout plan creation and management  
- Exercise library browsing  
- Workout logging  
- Progress tracking  

---

## Agile Delivery Structure

The project is structured into short, iterative sprints:

- Sprint length: 2 days  
- Sprint capacity: 8 story points  
- Planned work: 6 story points  
- Buffer: 2 story points for unplanned work  

This approach allows for rapid iteration while accounting for variability in AI-assisted development.

## Key Learnings

- Clear acceptance criteria significantly improve the quality of AI-generated code  
- Short sprint cycles enable rapid iteration and validation  
- Maintaining traceability between requirements and implementation improves clarity and organization  
- Structured Agile practices can be effectively applied even in solo, AI-assisted development environments
